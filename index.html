적용 방법

폰트를 별도로 설정하지않을 경우, 브라우저에서 지원하는 기본 서체로 표현된다.
CSS에 font-family속성이 있을 경우, 선언된 순서대로 좌에서 우로 이동하며 서체명을 검색해
디바이스에 해당 서체파일이 있으면, 표현하고 없으면 다음 서체명을 탐색한다.

iOS 5.1이상 부터 한글 기본폰트가 apple Gothic -> apple SDGothic NEO로 변경되면서
iOS 에서도 한글 bold체 지원이 가능해졌다.
iOS 7에서는 UI 기능상 다양한 글꼴 굵기가 들어가게 되면서 글꼴종류가 6가지로 늘어났다.

안드로이드의 웹앱에서 간헐적으로 한글 서체를 영문 기본서체로 잘못 표현하는 경우가 있다
이를 방지하기 위해 아래와 같이 기본 서체를 설정한다. 

                    font-family:droid sans,Dotum,"돋움",Helvetica,sans-serif;

폰트 단위 필수

px : 폰트 사이즈(font-size) 및 자간(letter-spacing), 들여쓰기(text-indent) 에서 사용

이슈사항

em 단위는 유지보수가 어렵고, 아이폰4(IOS 4)의 IR 기법 사용시 em으로 처리할 경우 성능이 현저히 느려지는 이슈가 있으므로
행간 외에는 사용을 지양한다.

행간(line-height)에 px단위를 사용할 경우, 안드로이드 기기 설정으로 CSS와 상관없이 폰트 크기가 커졌을 경우 대응할 수 없으므로,
반드시 em단위를 사용한다.

IS/IR 기법

IS (Image Sprite)기법

조각난 이미지(sprite image) 파일들을 하나의 파일로 병합 후 배경으로 처리해서 웹 문서 전송 속도를 높이는 기법

의미가 있는 이미지는 전경에 상응하는 텍스트를 포함시켜서 접근성을 확보해야 하기 때문에 IR(Image Replacement)기법을 함께 사용해야 함.

IR(Image Replacement)기법

의미가 포함되어 있는 이미지를 배경으로 처리(sprite background image) 하고 전경에 상응하는 텍스트를 넣는 방법.

접근성을 떨어트리지 않고 검색엔진으로부터 높은 가중치를 받을 수 있다.

스프라이트 이미지 제작 방법

기본 좌우 10px 간격으로 배치 한다.

가로 세로축 모두 1000px 이하로 제작한다.

스프라이트 이미지에 추가 이미지를 적용해야할 경우, 세로축으로 추가한다.
백그라운드 사이즈를 가로DP사이즈 * auto인 경우 수치를 수정해주지 않도도 되기 때문이다.

적용방법

박스모델

기본적인 박스 모델을 box-sizing : content-box로 설정되어 있으나,
모바일 n-screen 환경의 flexible 레이아웃 구현에는 부적합할 수 있어,
기본 박스 모델 설정을 box-sizing : border-box 로 변경하도록 한다. 

        
                    *, *:after, *:before{box-sizing:border-box;-webkit-box-sizing:border-box;-moz-box-sizing:border-box;}

CSS3 선택자(selector)

모바일에서는 CSS3 선택자를 광범위하게 지원하므로 사용에 큰 제약이 없다.

type선택자를 사용은 지양한다.

  • 일반, 인접 자손 선택자
  • 가상 요소(pseudo element)
    ::after, ::before
  • 가상 선택자 (pseudo class selectors)
    • :firts-child, :last-child, :nth-child(n)
  • 형제 선택자 : ~
  • 인접 형제 선택자: +

중첩 규칙

CSS는 정의 단계를 2 depth 이하로 권장한다.

html DOM 중첩은 기능영역이 5단계 이하를 권장한다.
의미가 없고, 스타일을 위한 레이어는 :after, :before 등의 가상선택자를 활용하여 DOM개수나 중첩을 줄일 수 있다.

OOCSS 에 따라 공통 규칙은 별도의 클래스로 분리한다.

하위 호환, 크로스 브라우징

하위 호환이 필요한 CSS 규칙에는 vendor prefix를 사용한다. 기본적으로 -webkit 과 -moz 까지 적용해 준다.

장비 특성별로 지원범위와 적용방법이 다를 수 있음을 유의한다.

vendor prefix를 반드시 사용해야하는 규칙들은 다음과 같다

  • border-radius
  • background 옵션들
  • box-sizing
  • flex box
  • gradient
  • Css Transform,CSSAnimation
  • box-shadow
  • text-shadow

background 속성들 상세정보

background position, attachment, background-size, background-origin, background-clip

백그라운드 속성들 중 color, url, repeat를 제외한 나머지 속성들은 축약형으로 제공하지 않는다.

background size를 축약형은 iOS6이하 안드로이드 4.3이하에서 지원하지 않는다.

background size와 offset position 조합은 iOS6이하에서 지원하지 않는다. 

        /****** background-size *******/
        background-size:50px 50px; /* cover|contain */
        -webkit-background-size:50px 50px;
        -moz-background-size:50px 50px;

        /****** background-clip :border-radius 적용영역의 배경의 잘림영역 설정 *******/
        background-clip:padding-box;
        -webkit-background-clip:padding-box;
        -moz-background-clip:padding-box;

        /****** background-orgin :배경이 적용되는 영역 설정 *******/
        background-clip:content-box;
        -webkit-background-clip:content-box;
        -moz-background-clip:content-box;
box-sizing 상세정보

        /* box-sizing */
        box-sizing:border-box;
        -webkit-box-sizing:border-box;
        -moz-box-sizing:border-box;

-webkit-transform 상세정보

CSS3의 -webkit-transform 을 이용한 애니메이션은, 스크립트를 이용한 애니메이션과 비교해 다음과 같은 장점이 있다.

1) 자바스크립트를 모르더라도 간단하게 애니메이션을 만들 수 있다.

2) 자바스크립트를 이용한 애니메이션은 잘 만들어졌더라도 성능이 좋지 못할때가 있다.
CSS3의 -webkit-transform을 이용할 경우, 하드웨어 GPU 가속을 사용할수 있으며, composite Layer 를 구성하여 렌더링 하므로 애니메이션 성능이 향상된다.

CSS3 2D Transforms, CSS3 3D Transforms 모두 성능개선 효과가 있다.

애니메이션 속성을 다음과 같이 대체하여 사용한다. left => translateX
top => translateY
filter:fade => opacity

2) 간헐적으로 transform이 position:fixed 과 함게 사용될 때 해당 영역 내의 요소가 깨질수있다.

다중 말줄임

웹킷엔진을 사용하는 브라우저에서는 display: -webkit-box; 속성과 -webkit-line-clamp 프로퍼티를 활용하여
여러줄의 문단도 CSS를 통해 말줄임 처리를 할 수 있다.

멜론 지원 범위를 커버할 수 있으므로 사용을 권장하되,
지원되지 않는 브라우저에 대한 처리를 반드시 함께 제공한다. (주의1참고)

주의 1

웹킷엔진을 사용하지 않는 브라우저에서는 해당 코드가 무시되기 때문에, 높이값을 지정하여 잘림 처리해 주어야 한다.
이때 line-height 값을 설정하고, 그에 비례하는 height 값을 주어야 한다.


예를들어, line-height: 14px; -webkit-line-clamp: 3;일 경우 height는 14px와 3을 곱한 42px 값을 준다.

line-height: 1.2em; -webkit-line-clamp: 2; 라면 height 값은 1.2em과 2를 곱한 2.4em을 주면된다.

Bug Report

fixed position 관련

초창기 모바일 폰은 지금보다 훨씨 크기가 작았기 때문에 fixed position을
아예 지원을 하지않거나 지원하더라도 버그가 상당히 많았는데,
모바일 환경이 많이 발전한 현재까지도 관련 버그가 남아있는 경우가 있다.

1) 입력박스 포커싱 시 하단으로 떨어짐

재현 환경 :아이폰 모든버전 webview

현상 : fixed 레이어 내부의 입력박스가 있을 경우, 화면 스크롤 후 입력박스에 포커싱할때 fixed 레이어 위치가 화면 하단으로 떨어지는 버그 있음<

대응방법 : focus in 이벤트 발생시 scrolltop을 강제로 최상단으로 이동시킨다.

2) 리사이즈 오류

재현 환경 : 안드로이드 모든버전 webview

현상 : window resize 이벤트 시 간헐적으로 fixed 영역이 깨지는 버그 있음

원인 : Android 버그이며 알려진 버그픽스 방법이 있으나,케이스에 따라 픽스할수 없는 경우도 있음

대응방법 :또는 fixed 대신 다른 relative를 적용한뒤, 약간의 delay time이후 스크립트로 fixed 속성으로 강제 변경시켜주어 해결할 수 있다
참조:http://catntuna.tistory.com/139

3) 하단영역링크 터치 오류

재현환경 : 안드로이드 젤리빈이하의 webview

현상 : 레이어팝업 위에서 하단레이어의 링크가 터치됨

원인 : 기기버그로 알려짐

대응 방법 : 모달레이어팝업의 DP할때 바닥레이어 또는 바닥페이지내 링크,입력박스 등을 hidden 시키도록 처리한다

4) 스크롤시 불안정함

재현 환경 : 안드2.3 이하 버전 기본 브라우저

현상 : 화면 스크롤시 fixed 영역이 위아래로 흔들림

원인 : Android 버그이며 픽스 불가능

5) 이미지 blur 오류

재현 환경 : Android 4.1.x 기본 브라우저

현상 : fixed영역 내의 이미지가 흐리게 blur 처리되는 문제.
border도 함께 흐려짐. retina display로 이미지를 축소하였을 때 더 잘 나타남.

원인 : Android 특정 버전 기본 브라우저의 렌더링 이슈

대응방법 :

  • text나 img가 인접요소 이면 블러 현상이 일어나지 않는다
  • position:fixed 인 영역에 img 태그 단독 사용시
    img태그에 position:relative나 position:absolute 속성을 적용하면 현상을 막을 수있다
  • position:fixed 인 영역에서 좌우 스와이핑이 일어날 경우, 나열되는(li) 태그들에 display:inline-block 속성을 적용하면 현상이 일어나지 않는다.
  • img태그에 opacity:1미만을 부여하면 블러 현상이 일어나지 않는다

iOS click이벤트 오류

현상 : 아이폰 iOS에서 a 링크가 아닌 요소에 click 이벤트를 bind 할 경우에 동작하지 않는 현상이 있음.

해결방법 :
1) 해당 요소에 cursor: pointer 를 적용할 경우
2) touch.js 를 이용한다.
